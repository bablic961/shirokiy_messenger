# Push-уведомления для PythonAnywhere

## Проблемы и решения

### 1. Требования для работы push-уведомлений

Push-уведомления работают **только через HTTPS**. PythonAnywhere предоставляет HTTPS по умолчанию для всех доменов `*.pythonanywhere.com`.

### 2. Текущая реализация

Система push-уведомлений состоит из:

- **Service Worker** (`static/sw.js`) - обрабатывает push-события
- **JavaScript клиент** (`static/push-notifications.js`) - управляет подписками
- **Backend** (`messenger/push_notifications.py`) - отправляет уведомления через pywebpush
- **VAPID ключи** - для аутентификации push-сообщений

### 3. Настройка на PythonAnywhere

#### Шаг 1: Установка зависимостей
```bash
pip install pywebpush==1.14.0 py-vapid==1.9.1
```

#### Шаг 2: Генерация VAPID ключей
```bash
python generate_vapid_keys.py
```

Это создаст файл `.env` с ключами:
```
VAPID_PUBLIC_KEY=...
VAPID_PRIVATE_KEY=...
VAPID_ADMIN_EMAIL=admin@yourdomain.com
```

#### Шаг 3: Настройка переменных окружения в PythonAnywhere

В веб-консоли PythonAnywhere добавьте переменные окружения:
1. Перейдите в Web tab
2. В разделе "Code" найдите "Environment variables"
3. Добавьте переменные из `.env` файла

#### Шаг 4: Обновите ALLOWED_HOSTS

В `settings.py` убедитесь, что добавлен ваш домен:
```python
ALLOWED_HOSTS = ['yourusername.pythonanywhere.com', 'localhost', '127.0.0.1']
```

#### Шаг 5: Collectstatic

```bash
python manage.py collectstatic --noinput
```

#### Шаг 6: Перезагрузите приложение

В веб-консоли PythonAnywhere нажмите "Reload yourusername.pythonanywhere.com"

### 4. Проверка работы

1. Откройте приложение через HTTPS: `https://yourusername.pythonanywhere.com`
2. При первом входе браузер должен запросить разрешение на уведомления
3. Откройте консоль браузера (F12) и проверьте:
   - Service Worker зарегистрирован
   - Push subscription создана
   - Нет ошибок в консоли

### 5. Тестирование

Отправьте сообщение другому пользователю. Push-уведомление должно прийти, если:
- Пользователь разрешил уведомления
- Пользователь не находится в активном чате
- Service Worker зарегистрирован

### 6. Известные ограничения

- **HTTP не поддерживается** - только HTTPS
- **Safari на iOS** - требует добавления сайта на домашний экран
- **Фоновая работа** - зависит от браузера и ОС
- **Батарея** - браузер может ограничивать фоновые процессы

### 7. Отладка

Если уведомления не работают:

1. Проверьте консоль браузера на ошибки
2. Убедитесь что используется HTTPS
3. Проверьте что разрешение на уведомления дано
4. Проверьте логи сервера на ошибки pywebpush
5. Проверьте что VAPID ключи загружены правильно

### 8. Альтернативные решения для PythonAnywhere

Если push-уведомления не работают, можно использовать:

1. **WebSocket уведомления** (Django Channels) - требует дополнительную настройку
2. **Polling** - периодические AJAX запросы для проверки новых сообщений
3. **Email уведомления** - для важных событий

### 9. Автоматическая инициализация

Система автоматически:
- Регистрирует Service Worker при загрузке страницы
- Запрашивает разрешение на уведомления через 2 секунды
- Отправляет подписку на сервер
- Обрабатывает входящие push-события

Код инициализации находится в `templates/messenger/base.html` (строки 343-357).

### 10. API эндпоинты

- `GET /api/push-subscription/vapid-public-key/` - получить публичный ключ
- `POST /api/push-subscription/save/` - сохранить подписку
- `POST /api/push-subscription/delete/` - удалить подписку

Все эндпоинты требуют аутентификации.
