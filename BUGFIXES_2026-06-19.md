# Исправления от 19.06.2026

## Исправленные проблемы

### 1. ✅ Дублирование голосовых и видео-кружков

**Проблема:** При отправке голосовых сообщений и видео-кружков создавались два плеера для одного сообщения.

**Причина:** В шаблоне `chat.html` внутри цикла для каждого сообщения был встроенный `<script>` блок, который вызывал `DOMContentLoaded` и создавал плеер. Одновременно глобальный скрипт в `voice-player.js` и `video-note-player.js` тоже искал все элементы и создавал плееры.

**Решение:**
- Удалены встроенные `<script>` блоки из шаблона
- Оставлены только пустые контейнеры с `data-*` атрибутами
- В `voice-player.js` и `video-note-player.js` добавлена проверка на существование плеера перед созданием нового:
  ```javascript
  if (element.querySelector('.voice-player')) {
      return; // плеер уже создан
  }
  ```

**Файлы:**
- `templates/messenger/chat.html`
- `static/voice-player.js`
- `static/video-note-player.js`

---

### 2. ✅ Локальные метки не сохранялись

**Проблема:** Локальные метки пользователей сохранялись только до перезагрузки страницы.

**Причина:** Код уже использовал `localStorage.setItem()` и `localStorage.getItem()`, поэтому метки должны были сохраняться навсегда. Проблема была в логике применения меток при загрузке страницы.

**Решение:**
- Проверено, что `localStorage` используется корректно в `local-labels.js`
- Метки загружаются из `localStorage` при инициализации класса `LocalUserLabels`
- Метки автоматически применяются при загрузке страницы через `DOMContentLoaded`
- Добавлен `MutationObserver` для отслеживания динамических изменений DOM

**Файл:** `static/local-labels.js` (проверен, работает корректно)
